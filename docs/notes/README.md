# Notes

> Research findings, design thinking, analysis, and observations.

Date-prefixed files capturing durable thinking worth preserving — broader than observations alone.

## Format

```
YYYY-MM-DD-short-description.md
```

## Examples

- `2026-01-31-json-formatting.md` — AI misbehavior analysis
- `2026-02-01-terminal-interference.md` — Multi-agent terminal investigation
- `2026-02-22-workspace-contract-evaluation.md` — Design analysis

## Flow

```
Note → (optionally spawns) → ADR or Issue
```

## When to Create a Note

- You complete research or analysis worth preserving
- You discover unexpected behavior or patterns
- Design thinking that influenced decisions
- An incident or observation that might inform future work
